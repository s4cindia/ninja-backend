Implement US-PDF-1.1: PDF Parser Service

Create a service that parses PDF files and extracts structure, metadata, and content for accessibility auditing.

File to create: src/services/pdf/pdf-parser.service.ts

Requirements:

1. Use pdf-lib for structure/metadata and pdfjs-dist for content extraction

2. Extract PDF metadata:
   - title, author, creator, producer
   - pdfVersion, pageCount
   - isTagged, hasStructureTree
   - language, creationDate, modificationDate

3. Extract structure information:
   - Document structure tree (if tagged)
   - Heading hierarchy
   - List structures
   - Table structures
   - Figure/image locations

4. Extract content per page:
   - Text content with positions
   - Images with dimensions and alt text (if present)
   - Links and their destinations
   - Form fields

5. Return type PdfParseResult:
   - metadata: PdfMetadata
   - pages: PdfPage[] (pageNumber, content, images, links)
   - structureTree: PdfStructureNode[] (optional)
   - isTagged: boolean

6. Handle errors gracefully:
   - Corrupted PDFs
   - Password-protected PDFs
   - Very large PDFs (implement streaming if needed)

Reference: Look at src/services/epub/epub-parser.service.ts for patterns.

Tests: Create src/services/pdf/pdf-parser.service.test.ts with unit tests using sample PDFs.

Install required packages if not present: npm install pdf-lib pdfjs-dist
